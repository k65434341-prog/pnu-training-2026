## 4. 페이지 구성요소

### 4.1 index.html (메인)
1. **헤더/네비** — 사이트 제목, 부산대 로고 자리, 2026 뱃지.
2. **히어로** — 큰 제목 + 소개 문구 + 요약 통계(전체 32개 과정 · 10개 분야 · 필수 N개). 통계는 시트에서 계산해 동적 표시.
3. **툴바(sticky)**
   - 통합 검색창 (과정명·한줄소개·상세소개 대상, 디바운스 200ms)
   - 필터: **분야 / 대상 / 이수구분 / 교육방식** (칩 또는 셀렉트) + 담당부서(2차)
   - 결과 개수 표시("32개 중 N개") + 필터 초기화 버튼
4. **과정 카드 그리드** — 카드 1개 = 과정 1개
   - 상단: 분야 칩(분야별 색), 이수구분 배지(필수 강조)
   - 제목: `과정명`
   - 요약: `한줄소개`
   - 메타 줄: `대상` · `교육방식` · `교육시간` · `신청기간`
   - 전체 카드 클릭 → `detail.html?code={과정코드}`
5. **상태 UI** — 로딩 스켈레톤 / 검색결과 없음(empty) / 로드 실패(에러 + 재시도).
6. **푸터** — 출처(부산대 인재개발원), 문의, 데이터 갱신 안내.

### 4.2 detail.html (상세)
1. **헤더/네비** — "← 목록으로" (index.html 복귀).
2. **상세 히어로** — 분야 칩 + 이수구분/교육방식 배지, `과정명`, `한줄소개`, `과정코드`.
3. **핵심 정보 카드(요약)** — 대상 · 이수구분 · 교육방식 · 교육시간 · 정원 · 교육기간 · 신청기간 · 담당부서 (라벨-값 그리드).
4. **본문 섹션**
   - 과정 소개 (`상세소개`)
   - 학습 목표 (`학습목표` → ` · ` 분리하여 리스트)
   - 주요 내용 (`주요내용` → ` · ` 분리하여 리스트/커리큘럼)
5. **신청 CTA** — `신청링크`로 이동하는 버튼(새 탭) + `문의`(부서·전화) 표시.
6. **상태 UI** — 로딩 / 잘못된·없는 `code`(안내 + 목록 복귀) / 로드 실패.

## 5. 기능 단위 분해 (Feature Breakdown)

### F1. 구글 시트 연동 (데이터 로더) — `app.js`
- `SHEET_URL` 상수(위 gviz URL) 정의.
- `loadCourses()`: `fetch(SHEET_URL)` → 텍스트 → `Papa.parse(text, {header:true, skipEmptyLines:true})` → 객체 배열 반환.
- 캐시 없음(항상 최신). 실패 시 `throw` → 호출부에서 에러 UI.
- **검증 포인트**: 32개 행, 각 행에 17개 키, `과정코드[0] === 'PNU-2026-001'`.

### F2. 메인 렌더링 & 라우팅
- 로드된 배열을 카드 그리드로 렌더. 카드는 `<a href="detail.html?code=...">`.
- 상세는 `new URLSearchParams(location.search).get('code')`로 코드 취득 → `courses.find(c => c['과정코드'] === code)`.

### F3. 검색 (Search)
- 입력값을 소문자·공백 정규화 → `과정명 + 한줄소개 + 상세소개` 결합 문자열에 부분일치.
- 디바운스(200ms). 필터와 **AND** 결합.

### F4. 필터 (Filter)
- 팩싯: `분야`, `대상`, `이수구분`, `교육방식` (+ `담당부서` 2차).
- 각 팩싯 옵션은 **데이터에서 distinct 추출**(하드코딩 금지) → 시트에 값이 추가돼도 자동 반영.
- 다중 팩싯은 AND, 동일 팩싯 내 다중 선택은 OR(칩 방식 채택 시).
- "필터 초기화"로 전체 리셋. 결과 개수 실시간 갱신.
- (선택) 필터 상태를 URL 쿼리에 반영해 공유 가능하게.

### F5. 상세 표시
- 요약 카드 + 본문 섹션 렌더. `학습목표`·`주요내용`은 ` · ` 기준 분할하여 목록화.
- 빈 값 컬럼은 해당 섹션 자동 숨김.

### F6. 상태·오류 처리 (공통)
- 로딩 스켈레톤 → 성공 시 콘텐츠 / 실패 시 재시도 버튼.
- 상세: `code` 누락·불일치 시 안내 + 목록 복귀 링크.

## 6. 디자인 방향

> `frontend-design` 스킬이 현재 세션에 설치돼 있지 않아, 아래 디자인 시스템을 직접 정의해 적용한다. (요청 시 스킬 설치 후 재적용 가능)

- **톤**: 공공기관 신뢰감 + 현대적 정보 카탈로그. 부산대 상징색(딥 블루/네이비) 기반.
- **색상 시스템** (CSS 변수)
  - Primary(네이비) `#123c7b`, Primary-strong `#0b2a5e`, Accent(골드) `#c8a45c`
  - 배경 `#f7f8fb`, 카드 `#ffffff`, 텍스트 `#1a2233` / 보조 `#5b6472`, 보더 `#e6e9f0`
  - 이수구분 배지: 필수 = 레드톤(강조), 권장 = 앰버, 선택 = 뉴트럴 블루
  - 분야 칩: 10개 분야에 접근성(AA) 만족하는 색 매핑
- **타이포그래피**: **Pretendard**(jsDelivr CDN) + 시스템 폰트 폴백. 제목 700, 본문 400/500.
- **레이아웃**: 최대폭 1120px, 8px 스페이싱 스케일, 카드 radius 16px, 부드러운 그림자.
- **카드 그리드**: `repeat(auto-fill, minmax(300px, 1fr))`, hover 시 살짝 떠오름.
- **반응형**: 모바일 우선. 모바일에서 필터는 접이식(details/드로어), 그리드 1열.
- **접근성**: 시맨틱 태그, `aria-label`, 키보드 포커스 링, 명도 대비 AA, `prefers-reduced-motion` 존중.
- **외부 의존성**: PapaParse + Pretendard만 CDN 사용(정적 사이트라 CDN 허용). 그 외 프레임워크 없음(바닐라 JS).
